Personal Project · 2026 · Cairo

NEXUS — Multilingual RAG & NER Intelligence System

LangChainQdrant Gemma4 12BQwen3 Embed GLiNERAraBERT LM StudioPython Re-rankingNER
GitHub (add link)
Situation

Industrial and financial document corpora in the Arab world are inherently mixed-language — the same entity appears as "Al Noor Cement", "Al-Noor Cement Co.", "شركة النور للاسمنت", and "شركة النور" across the same document set. Standard RAG pipelines and off-the-shelf NER models fail here: they can't resolve aliases across scripts, miss Arabic-specific morphology, and hallucinate when the retrieval layer doesn't distinguish person from organization from legal entity.

Task

Build a production-grade multilingual RAG system from scratch — one that indexes thousands of Arabic and English documents, understands entity aliases across both scripts, extracts typed entities (persons, organizations, facilities, legal IDs, contact info) with high precision, and serves grounded answers with source citations — all running locally on a laptop with no cloud dependency.

Action

Architected the full pipeline in LangChain — ingesting PDF, TXT, and MD documents, chunking with overlap-aware strategies, and embedding via Qwen3 Embed (served locally via LM Studio) for dense vector representations stored in Qdrant. Plugged in a re-ranker stage before generation to sharpen retrieval precision on ambiguous cross-language queries.

Built a dual-NER pipeline: GLiNER handles English and European-language documents — extracting persons, organizations, facilities, legal entities, contact details, and IDs with span-level precision. AraBERT (trained on 80 GB of Arabic data) handles Arabic documents — correctly resolving name suffixes, script variants, and morphological forms that Latin-script models miss entirely. Both models run in parallel, surfacing typed entity chips with source citations.

The LLM layer runs Gemma4 12B locally via LM Studio — no API calls, no data leaving the machine — with full system status visible in a live sidebar panel.

Result

Indexed 61,351 vectors across 2,912 files in a single Qdrant collection. Stress-tested on 3,000 files running fully locally. The system correctly resolves entity aliases across Arabic and English scripts — ask in English about a person whose records are in Arabic and it finds them, and vice versa. When asked for a field not present in the documents, the system explicitly cites the source document confirming absence — no hallucination, grounded refusal with evidence.

Academic Project · McMaster University · 2024

Student ID Seven-Segment Display

Digital Logic Design K-maps NAND-only Minimization Circuit Simulation Breadboard Implementation Seven-Segment Display
GitHub (add link)

Live demo — breadboard

Situation

The goal was to display every digit of a student ID in sequence on a single seven-segment display — no microcontroller, no code. Pure combinational and sequential logic built from discrete gates on a breadboard.

Task

Design the complete digital logic circuit from scratch: derive truth tables for each digit's segment encoding, minimize using Karnaugh maps (K-maps), implement exclusively with NAND gates to minimize transistor count, simulate to verify correctness before touching hardware, then build and demonstrate on a physical breadboard.

Action

Derived the Boolean expressions for all seven segments (a–g) across each digit of the student ID. Applied K-map minimization to reduce gate count and simplify logic. Converted all expressions to NAND-only form using De Morgan's law — NAND gates use fewer transistors than AND+NOT combinations and are the standard cell in real CMOS circuits, making this a genuine engineering choice, not a constraint.

Simulated the complete circuit in an online circuit simulator to verify segment sequencing and catch logic errors before committing to hardware. Once simulation passed, transferred the design to a breadboard with physical ICs, wired all gate connections manually, and verified each digit displayed correctly in sequence.

Result

A working physical circuit that sequences through every digit of the student ID on a single seven-segment display — designed, simulated, and built end-to-end using only combinational logic. Demonstrates practical knowledge of Boolean algebra, K-map minimization, NAND universality, and the simulation-to-hardware workflow.

Academic Project · McMaster University · 2025

Ball-and-Beam PID Control System

MATLABSimulink QUARCNI myRIO PID / PD ControlRoot Locus
GitHub (add link)
Situation

The ball-and-beam system is a classic unstable control problem — a ball balancing on a beam whose angle is controlled by a motor. Without active feedback, the ball rolls off. The challenge: model the nonlinear electromechanical dynamics accurately enough that a linear controller works on real hardware.

Task

Derive the plant transfer function from first principles, design PD and PID controllers using root-locus methods, implement them in Simulink with QUARC, and validate performance on a physical NI myRIO-controlled rig — measuring settling time, overshoot, disturbance response, and steady-state error.

Action

Modeled the full electromechanical dynamics of the beam actuator and ball position. Derived the open-loop transfer function and plotted root locus to identify stable controller regions. Designed a PD controller first to achieve fast settling, then added integral action to eliminate steady-state error under disturbance. Deployed via QUARC real-time target on NI myRIO and ran live experiments measuring step response, disturbance rejection, and parameter sensitivity.

Result

Achieved stable ball positioning with controlled settling time and minimal overshoot on the physical hardware — validating the complete design loop from mathematical model to real-time controller on embedded hardware.
